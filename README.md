# Automatic Detection & Attribute Classification of Maltese Traffic Signs

This repository contains the implementation, experiments, and results for the ARI3129 – Advanced Computer Vision for AI group project. The project focuses on the automatic detection of Maltese traffic signs and the classification of selected sign attributes using modern computer vision techniques.

## Notebook Descriptions

### 2a_model_name.ipynb
Implements and trains the selected object detection model for traffic sign localisation and classification.  
Includes dataset loading, model training, evaluation using standard detection metrics (e.g. mAP, precision, recall), and qualitative visualisation of results.

### 2b_model_name.ipynb
Implements the attribute classification model for a selected traffic sign attribute (e.g. viewing angle).  
Includes sign cropping, classifier training, evaluation using classification metrics, and error analysis.

---

## Utility Scripts

### coco_crop_dataset.py
A PyTorch Dataset class that loads COCO-format annotations and automatically crops individual traffic sign instances based on their bounding boxes.  
Used by some notebooks for attribute classification tasks to extract sign regions from full images for training and evaluation.

---
## Dataset (CS2 Dataset/)

The consolidated dataset includes images captured on Maltese streets and annotated using Label Studio after being processed with `LS2COCO.ipynb` and `MemberMerger.ipynb`, with separate YOLO and COCO format splits for each attribute.

Note: The dataset was not uploaded to this repository due to GitHub file size limits. The complete dataset can be found on [Google Drive](https://drive.google.com/drive/folders/1bPxVoXl6eLSzq9qq4dz779dkLnRC_sOA?usp=drive_link).

## Results

### Group Results/
Contains aggregated performance summaries across all evaluated models in CSV format (generated automatically by the notebooks and manually edited for consistent formatting). These results are loaded and used in `2c_results_comparison.ipynb`.
Includes metrics such as mAP@0.5, mAP@0.5:0.95, precision, recall, and inference speed for cross-model comparison and analysis.
### Individual Results/

Stores detailed outputs generated by each script during model training and evaluation, including training and inference logs (runs/), visualisations such as loss curves and precision–recall curves (plots/), and saved model checkpoints (weights/).  
This directory also contains some prerequisites (e.g. modified dataset versions) required by certain notebooks.

### Models/

The Models directory is where trained model weights will be automatically generated and saved during training. They can also be accessed through the provided [Google Drive link](https://drive.google.com/drive/folders/1bPxVoXl6eLSzq9qq4dz779dkLnRC_sOA?usp=drive_link).

## Getting Started

Clone this repository and install required dependencies using `pip install -r requirements.txt`. Place your dataset in `CS2 Dataset/` and run the desired notebook (`2a_model_name.ipynb` or `2b_model_name.ipynb`). Results will be automatically saved to `Individual Results/` and aggregated in `Group Results/`. Final models will be saved in `Models/`.
